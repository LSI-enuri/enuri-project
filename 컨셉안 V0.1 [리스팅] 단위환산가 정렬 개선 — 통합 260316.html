최종작성일2026.03.16
기획담당자에누리서비스기획 이상인
01 프로젝트 개요
범위
PC / MW / APP
적용 페이지
카테고리 페이지 (LP), 검색결과 페이지 (SRP)
목적
단위별 분산된 정렬을 통합하여 고객이 단위 선택 없이도 단위환산가가 있는 상품 중 실제 가장 저렴한 상품을 바로 탐색
02 현황 (As-Is) vs 개선 (To-Be)
As-Is — 현황
단위별 분리 정렬 (ml당 / g당 / L당 ...)
· 고객이 단위 직접 선택 필요
· 다른 단위의 더 저렴한 상품 미노출
· 최적 가성비 상품 발견 실패 위험
· 정렬 UX 파편화
→ 고객 구매 기회 손실
To-Be — 개선
단위 통합 정렬
· 단위 선택 불필요
· 전 상품 단위환산가 기준 통합 노출
· 실질적 최저가 상품 최상단 노출
· 직관적 정렬 UX
→ 단위환산가가 있는 모델의 최저가 통합 탐색 가능
03 필수 정책
P1
정렬명: 단위환산 최저가순 (UI 표기 동일)
P2
단위환산가 기준 오름차순 정렬 (단위 종류 무관, 전체 통합)
P3
단위환산가 없는 상품 → 정렬 대상에서 제외
P4
모델에 속한 상품들 중 단위환산 최저가 상품을 기준으로 정렬 및 대표모델이 됨
P5
정렬 제공 범위: 단위환산가가 있는 가격비교 모델
(PC: 전체, 가격비교 탭에서만 정렬 노출 / MW·APP: 전체 리스트)
P6
단위 업데이트 주기: 카테고리 재분류 및 새 단위 생성 시 24시간(1일) 주기로 업데이트 후 프론트 반영
* 현재 정책 유지
참고
단위환산가 산출 기준 (가격 기준)
PC — 배송비포함 필터 선택 전: 배송비 미포함 최저가 기준
PC — 배송비포함 필터 선택 후: 배송비 포함 최저가 기준
MO — 배송비 포함 최저가 데이터 없으므로 데이터 추가 이후 보완 예정
04 동률 처리 정책 (Tie-breaking) — 제안
케이스 A — 동일 단위 + 동일 단위환산 최저가
케이스 B — 다른 단위 + 동일 단위환산 최저가
구분 정렬 우선순위 비고
1안 (LP/SRP) 인기순 높은 상품순 > 동률 시 랜덤 · 인기순 상품 우선 노출
· 단위환산 최저가순 정렬 내 인기순 정렬 활용 가능한지 개발 검토 필요
2안 (LP/SRP) 단위환산 최저가 상품의 최저가 가격순 > 동률 시 랜덤 구현 단순, 가격 직관성 높음
3안 (LP) 랜덤
(SRP) 상품이 많은 카테고리순 > 같은 카테고리 시 랜덤		 LP/SRP 로직 분리 > 개발·운영 복잡도 증가
